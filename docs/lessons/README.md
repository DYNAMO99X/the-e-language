# E — A Friendly Curriculum

A 16-lesson course for absolute beginners (ages ~10 and up) who want to
learn to code. The language is E, but the *thinking* you learn here is
the same thinking Python, JavaScript, and every other language uses.

## Who this is for

- A teacher or parent working with a student
- A self-learner who wants a structured path
- Someone who tried Scratch and is ready for text-based coding but
  doesn't want the syntax tax of a "real" language yet

## How to use it

For each lesson, open two things side by side:

1. The **teacher's guide** (`NN_topic.md`) — read this first. It has the
   goal, vocabulary, talking points, and challenges.
2. The **starter program** (`NN_topic.e`) — the student types this
   exactly, runs it, then starts changing it according to the
   challenges in the teacher's guide.

Total time: about 12–18 hours of instruction, depending on pace.

## After this

When the student is comfortable with all 16 lessons, they're ready to
move on to a "real" language. E is designed as a stepping stone between
visual block programming (Scratch) and text programming. The
[bridge_to_python.md](../bridge_to_python.md) doc shows how E maps to
Python, so the next step is a small one.

## Teaching tips

1. **Run the code first.** Before explaining anything, type the program,
   run it, and let the student see what it does. The "wow" matters.
2. **Encourage tinkering.** Every lesson has a "Tinker" section. The
   student should change numbers, swap words, see what breaks. Break
   things on purpose.
3. **Read errors together.** Don't paper over errors. E's error messages
   are friendly — read them out loud, ask "what does this mean?",
   figure out the fix.
4. **Skip ahead is fine.** If a student already knows a topic, mark it
   done and move on. Each lesson is self-contained.

Have fun. Programming is supposed to be fun.
